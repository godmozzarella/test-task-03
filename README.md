# Уютная кухня — лендинг

Тестовое задание: вёрстка одностраничного лендинга кафе.  
Стек: Vite + чистый HTML / CSS / JS (без UI-библиотек).

## Запуск

```bash
npm install
npm run dev   # http://localhost:5173
npm run build # продакшен-сборка в /dist
```

## Структура

```
├── index.html        вся разметка страницы
└── src/
    ├── styles.css    все стили (один файл, секции по блокам)
    ├── main.js       точка входа — вызывает init-функции
    ├── slider.js     логика карусели
    ├── accordion.js  логика аккордеона
    └── booking.js    отправка формы + управление модалом
```

## Приёмы и решения

### Адаптив
- Единственный брейкпоинт `767px` (мобайл ↔ десктоп).
- Мобильная минимальная ширина зафиксирована: `min-width: 375px`.
- Hero-изображение меняется через `display: none` — отдельный файл для каждого разрешения без JS.

### Типографика
- Базовый размер шрифта задан на `html` через `clamp(1rem, 0.833vw, 2rem)`: до 1920 px остаётся 16 px, выше - пропорционально растёт до 32 px на 4K.
- Заголовки используют `clamp()` для плавного масштабирования между мобайлом и десктопом.
- Все размеры в `rem` — автоматически подхватывают масштаб `html`.

### CSS-переменные
Все цвета, радиусы и служебные значения вынесены в `:root`. Ни одного сырого цвета в компонентных правилах нет.

### Карусель
- Чистый JS без библиотек: `flex`-трек + `translateX` через `calc`.
- На десктопе — 3 слайда, сдвиг на 1; на мобайле — 1 слайд, стрелки поверх изображения.
- Точки-индикаторы синхронизированы с активным (левым) слайдом.

### Аккордеон
- Использует нативный атрибут `hidden` и `aria-expanded` — корректно для скринридеров.
- Индикатор `+` / `-` обновляется через JS.

### Форма и модал
- Валидация через `checkValidity()` / `reportValidity()` — браузерная, без кастомных библиотек.
- Модал управляется атрибутом `hidden`; закрывается по клику на фон (`data-modal-close`) или кнопку.
- Связь формы и модала — прямая передача ссылки, без глобального состояния.

### Семантика
`header`, `main`, `section`, `nav`, `form`, `button` — везде по назначению. Интерактивные элементы — нативные `<button>` и `<a>`, доступны с клавиатуры без доп. атрибутов.
